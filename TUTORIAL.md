# byteworker 上手引导(tutorial)

> **给 AI 助手**:用户**首次使用** byteworker、且同意走引导时,你读本文件,按下面的
> 剧本对话式地带用户走完。**读一次、记住剧本即可,引导途中不要反复重新加载本文件。**
> 用户随时说「跳过 / 不用了 / 直接开始」→ 立刻结束引导、转入正常使用
> (第 1 步的建库若还没完成,仍需完成 —— 否则知识库无处可存)。

引导目标:用 1-2 分钟让用户**亲手走一遍** byteworker 的核心闭环 ——
建知识库 → 摄取一篇文档 → 查询一次 —— 从而理解整个流程。全程友好、简短,
一次只推进一步、等用户回应再继续;别堆术语。

---

## 剧本

### 开场(一段话)

告诉用户:byteworker 把你日常的飞书文档 / 会议 / 群聊**消化**成一个
**可对话查询的知识库**;这个引导会带你亲手做三件事 —— 建库、摄取一篇文档、查询一次 ——
大约 1-2 分钟。随时说「跳过」就直接开始用。下面的子命令都是用类似`/byteworker [子命令] [参数]`的方式使用。

### 第 1 步 · 创建你的知识库

1. 解释(两句):知识库数据存在一个**你指定的独立目录**里 —— 不在 skill 仓库内,
   含机密、绝不外传;它本身也是个本地 git 仓库,每步操作可回滚。
2. 问用户:把知识库数据目录放在哪个父目录下?(目录名默认 `byteworker_kb`,可改)
3. 按 DESIGN.md §1.B 初始化该目录,把绝对路径写进 skill 的 `.kbconfig`。
4. 告诉用户:知识库建好了,在 `<路径>`;以后所有笔记都沉淀在这里。

### 第 2 步 · 摄取第一篇文档

1. 解释(两句):「摄取(digest)」= 把一份资料**消化**成结构化笔记 —— 不只是存档,
   是抽取里面的决策、结论、人物、项目,沉淀成相互链接的节点。
2. 要一份输入:
   - 用户在**首条消息里已经给了**飞书文档 / 妙记 URL 或本地 md → 直接用它。
   - 否则请用户给一个飞书文档 / 妙记 URL,或一个本地 `.md` 文件路径。
   - 用户手头没有、或飞书还没配好 → **不要卡住**:告诉他可以稍后再摄取,本步先跳过,
     直接进第 3 步、用概念讲解代替演示。
3. 按标准 digest 流程摄取。完成后用**大白话**汇报:这篇资料消化出了哪些节点
   (例如「3 个人物、1 个项目、2 个决策、1 个事件」)、存到了哪、原文已逐字留底。
4. 点一句关键理念:同一个项目以后在别的文档 / 会议里再被提到,会**汇聚到同一个节点**
   持续生长 —— 这就是「实体图」,而不是一堆散落的文件。

### 第 3 步 · 查询知识库

1. 解释(一句):「查询(search)」= 用一句话问知识库,答案带**出处**和**置信度**。
2. 引导用户问一个问题:
   - 基于刚摄取的内容,给用户 1-2 个示例问题挑(如「关于〈某人〉我知道什么」
     「我们关于〈某主题〉定过什么」),也可让他自己问。
   - 若第 2 步跳过了摄取 → 讲清楚查询会怎么工作即可,不强行演示。
3. 按标准 search 流程查询,展示:答案 → 出处 → 置信度。点明**出处可回原文核对**。

### 第 4 步 · 设置个人context
1. 解释（一句）：「上下文(context)」= 有需要额外告诉agent的信息，需要agent注意的做事方法，以及任何不在知识库但是需要agent知道的信息都可以放到这里
2. 引导用户设置一个context：
   - 自己工作所在的部门的名字、工作范围，自己管理的团队，项目的范围
   - 自己的主要领导、下属是谁，提取信息的时候额外关注这些人
   - 其他不在输入的文档中，但是需要模型知道的信息
3. 展示最终的context内容

### 收尾 · 串起整个流程

1. 三句话总结闭环:**摄取**(digest)把资料消化进库 → 知识沉淀成**实体图**持续生长
   → 随时**查询**(search)一句话问出来、带出处。
2. 列一下还能做什么:`update` 更新某条知识 · `brief` 会前简报 · `dashboard` 工作看板 ·
   `help` 帮助;还能在 skill 目录运行 `bin/browse.sh` 在浏览器里可视化浏览整个知识库。
3. 告诉用户:引导结束,可以正常用了 —— 直接发飞书 URL 或用自然语言即可。
   想再看一遍引导,说「跑一下上手引导」。

## 注意事项(给 AI 助手)

- 全程对话式、简短、友好;**一次只做一步**,等用户回应再继续。
- 用户随时说「跳过 / 不用了」→ 立即结束引导、转正常使用;第 1 步的建库若没完成,
  仍需完成。
- 引导途中**不要反复 Read 本文件** —— 开头读一次即可。
- 第 2 步摄取走标准 digest 扇出与「写入规范」,第 3 步查询走标准 search 流程 ——
  引导只是「带着用户做一遍」,底层行为与平时完全一致,细节见 SKILL.md。
